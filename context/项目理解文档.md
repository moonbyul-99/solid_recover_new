本文档是对 `c:/Users/sunrui/Desktop/solid-recover-dev/` 原项目在重构启动时的全面分析，用于承载重构依据。重构目标代码位于同仓库 `sr_new/`。

## 1. 项目定位

Solid Recover 是一个 **单细胞多组学整合与跨模态预测** 的深度学习项目，核心能力：

1. 为每种组学（RNA、ATAC、Gene Activity 等）训练一个 VAE/AE 式编码器，得到低维共享表征
2. 通过对比学习（CLIP-style）和跨模态重建，把配对的不同组学投影到同一 latent 空间
3. 基于共享 latent 做跨模态生成（如 RNA → ATAC）、匹配评估（top-k hit / FOSCTTM / matchscore）、聚类、GRN 推断等下游分析

## 3. 数据流

### 3.1 配对训练（pair_scratch）

```
h5mu (train/test) 
  └─ muon.read_h5mu 
       └─ MuData[key_1], MuData[key_2]         # 两个 AnnData
            └─ Base_sr._adata_format           # 稀疏/稠密 → torch.float32 CPU tensor
                 └─ pair_data(Dataset)         # 仅保留 omic_1/omic_2 两个张量
                      └─ DataLoader
                           └─ SRPairVAE.forward(x1, x2)
                                ├─ model_1.get_embedding(x1) → z, z_mu, z_logvar, z_embed
                                ├─ model_2.get_embedding(x2) → y, y_mu, y_logvar, y_embed
                                ├─ self-recon: x1 = decoder_1(z_embed), x2 = decoder_2(y_embed)
                                ├─ cross-recon: x1_c = decoder_1(y_embed), x2_c = decoder_2(z_embed)
                                └─ VAE_clip_loss(...)
```

### 3.2 单组学预训练（single）

```
HuggingFace datasets.load_from_disk
  └─ train_test_split(seed)
       └─ 直接送 DataLoader （HF Dataset 支持 __getitem__ 返回 dict）
            └─ SRVAE.forward → recon + KL
```

### 3.3 配对微调（pair_pretrain）

与 pair_scratch 相同，仅在 model 构造后通过 `load_pretrained_model(omic_1_ckpt, omic_2_ckpt)` 加载单组学 ckpt 到 `model_1` / `model_2`。

## 4. 训练循环（现状）

现位于 `Base_sr.train_model`，职责混乱：

- 继承 `nn.Module`（只是为了调用 `.to(device)`）
- 自持 `train_loader` / `test_loader` / `optimizer` / `scheduler` / `writer` / `project_dir`
- 循环中硬编码 "最后一个 batch 样本数 < 0.8*batch_size 则 skip"（避免对比学习 batch 过小）
- 每步把 loss_dict 中所有含 'loss' / 'logit_scale' 的值写入 tensorboard

## 5. 损失公式

`VAE_clip_loss`：

```
L = cross_recon_1 * recon_cross_1 + (1-cross_recon_1) * recon_self_1
  + cross_recon_2 * recon_cross_2 + (1-cross_recon_2) * recon_self_2
  + vae_beta_1 * KL_1
  + vae_beta_2 * KL_2
  + clip_weight * CLIP(z_embed_1, z_embed_2)
```

CLIP 使用 `z_embed`（reparam 后），不用 `z_mu`（注释说 z_mu 效果不好）。

`WeightedCLIPLoss` 在 CLIP 基础上，对 top-k / bottom-k 非对角元素加权（log-sum-exp 数值稳定版），用于"困难负样本"加权。

## 8. 评估流（原 pair_eval.py）

1. 从 `output_dir/config.yaml` 重建模型
2. 遍历 `output_dir/models/ckpt_*.pth`
3. 每个 ckpt：
   - 计算 z_mu / y_mu（**一次性**放 GPU，可能 OOM）
   - 计算 top-k hit / acc / matchscore / foscttm
   - 画 UMAP（占位 `np.random.rand` 作为 .X 配合 `obsm['mu']` 做 neighbors）
   - 保存 `match_metric.json` 和 `embedding_umap.png`

## 9. pred_evaluation/ 和 result_analysis/ 中可整合的能力

- `pred_evaluation/get_sr_pred.py::sr_pipe`：按 batch 做交叉重建 → `h5ad` 保存。整合为 API + CLI `predict` 子命令
- `result_analysis/utils.py::get_sr_embed`：按 batch 提取 z / z_mu / z_embed。整合为 `evaluation/embeddings.py::get_embeddings`
- `result_analysis/utils.py::ckpt_merge`：合并多 ckpt `match_metric.json`。整合为 `evaluation/reporting.py::ckpt_sweep`
- `result_analysis/utils.py::compute_metric_chunked`：并行 AUPRC/AUROC。整合为 `evaluation/reporting.py::compute_metric_parallel`
- 可视化类脚本（`visualization.py` 等）：仅在 notebooks 中示例，不整合进库

## 10. checkpoint 兼容性

原 ckpt 格式：
```python
torch.save({'model_state_dict': self.model.state_dict()}, 'ckpt_xxxx.pth')
```

`self.model` 对应：
- `single_sr`: `sr_vae` 实例
- `pair_sr_scratch` / `pair_sr_pretrain`: `sr_pair_vae` 实例

重构时保持以下 state_dict 键不变：
- `encoder.encoder_header.*` / `encoder.fc_blocks.*`
- `mu_proj.*` / `logvar_proj.*`
- `decoder.decoder_header.*` / `decoder.fc_blocks.*`
- `model_1.*` / `model_2.*`（pair 场景）

这样新 `SRVAE` / `SRPairVAE` 可无缝加载旧 ckpt。
