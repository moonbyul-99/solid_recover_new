## Task 3：Trainer + 模型门面

### `training/trainer.py::Trainer`（新增）

独立训练器类，不继承 `nn.Module`。职责：

- 持有 `module: nn.Module`、`loss_fn: nn.Module`、`optimizer`、`scheduler`、`writer`
- `prepare_project_dir(project_dir)`：目录存在则追加 `_{timestamp}`，建 `models/` 与 `logs/`
- `fit(train_loader, test_loader, train_steps, eval_every, save_every, device, loss_step_fn)`：训练主循环，兼容单组学和配对场景
- `loss_step_fn(module, batch, loss_fn, device) -> (outputs, loss_dic)`：由模型门面注入，解决"单 batch 含 feature"vs"配对 batch 含 omic_1/omic_2"的差异
- 保留"最后一个 batch 样本数 < 0.8 * batch_size 则 skip"的原始行为

### `models/base.py::BaseModel`（新增，普通 Python 类）

- 持有 `module`、`loss_fn`（可为 None）、`optimizer_cfg`
- `setup_data(...)` / `configure_loss(...)` / `configure_optimizer(...)` / `train(...)` / `load_checkpoint(...)` / `from_pretrained(...)` 等通用方法
- 子类负责构造 `module` 和 `loss_fn`，以及提供 `_loss_step(batch, device)`

## Task 2：`SRPairVAE.forward` 输出统一 + `model_2` dropout

旧 forward 返回 `{x1: dict, x2: dict, x1_c_recon, x2_c_recon}`（cross-recon 被单独拎出去，命名还保留了 RNA+ATAC 偏见）。v2 改为：

```python
{
    "x1": {"x_recon": ..., "cross_recon": ..., "z", "z_mu", "z_logvar", "z_embed"},
    "x2": {"x_recon": ..., "cross_recon": ..., "z", "z_mu", "z_logvar", "z_embed"},
}
```

同步改动：

- `losses/composite.py::VAEClipLoss.forward`：读取 `x1_dic["cross_recon"]` / `x2_dic["cross_recon"]`
- `models/pair.py::_process_batch`：`flat_outputs` 键从 `x*_c_recon` 重命名为 `x*_cross_recon`（仍含 `recon` 子串，Trainer 的 batch-size 启发式兼容）
- `evaluation/prediction.py::predict_cross_modality`：改为消费新路径
- `nn/pair_vae.py::SRPairVAE.__init__`：`model_2` 显式转发 `dropout_p`（legacy 遗漏）；dropout 无可学习参数，旧 ckpt 仍可加载

## Task 3：`predict_cross_modality` 字段命名统一

| 旧键 | 新键 |
| --- | --- |
| `rna2atac` | `omic_2_cross_recon` |
| `atac2rna` | `omic_1_cross_recon` |
| `z_mu_1` / `z_mu_2` | 保留 |

`evaluation/prediction.py::predict_cross_modality` 返回值由 `{rna_pred, atac_pred}` 同步改为 `{omic_1_pred, omic_2_pred}`；`predictions_to_anndata` 的 `modality` 默认值相应改为 `"omic_1"`。notebooks 同步。

## Task 4：`adata_to_tensor` 简化 + `to_gpu` 文档化

- `data/adata_utils.py::adata_to_tensor`：删除 `sparse_chunk_size` 参数和 chunk + concat 分支，稀疏矩阵直接 `.toarray()`。旧 chunked 路径在 chunks list + 最终 dense 同时在内存中时峰值反而更高；切片后 <=~100k 行的 split 上单次 `.toarray()` 是更简单、峰值更低的实现。
- `data/datasets.py`：给 `PairDataset.to_gpu` 补 docstring 说明使用场景；`SingleDataset` 对称新增 `to_gpu` 方法。

## Task 5：`BaseModel.setup_data` 允许 `test=None` + config 持久化

- `BaseModel.setup_data(train_dataset, test_dataset=None, ...)`：`test_dataset` 改可选，省略时 `self.test_loader = None`；`train(...)` 前置检查只要求 `train_loader`；`Trainer.evaluate` 已经处理 `test_loader is None`，上层可直接跑。
- `BaseModel.train(...)` 新增 `training_config: Optional[TrainConfig]` 参数，并抽出 `_persist_config` 辅助方法：
  1. 有 `training_config` → 用 `dump_train_config` 写入有效 config（反映 in-memory 改动）
  2. 仅有 `config_copy_path` → `shutil.copyfile` 原 YAML（legacy 行为）
  3. 都没有 → warning，不写 config
  CLI 入口仍然走 `config_copy_path` 路径；notebook / 库用户推荐走 `training_config`。
